# patent-writing

一个用于中国发明专利技术交底书撰写的 Claude Skill，针对"分章节、逐步推进"的真实专利写作工作流深度定制。

---

## 功能简介

上传一份参考技术交底书，告诉 Claude 新专利的标题和方向，然后逐章推进——这个 skill 会让 Claude 严格对标示例的行文风格，产出专业、连贯、不 AI 化的专利文本。

覆盖技术交底书的完整六章结构：

| 章节 | 内容 |
|------|------|
| 第一章 | 背景技术与现有技术缺点（含专利引用） |
| 第二章 | 发明所解决的技术问题 |
| 第三章 | 技术方案详细阐述（含步骤 + 公式） |
| 第四章 | 与现有技术相比的优点 |
| 第五章 | 技术关键点和欲保护点 |
| 第六章 | 附图（Mermaid 流程图） |

---

## 使用方式

### 典型起手指令

```
现在我需要完成一篇新专利，格式与内容风格参考这个文档。
新专利的标题为：一种基于XXX的YYY方法
主要内容围绕：[技术方向简述]
注意：内容连贯，行文符合逻辑，不口语化也不 AI 化；内容真实，不要无关杜撰；严格参考示例的写法和行文风格
好的，现在开始写：一、介绍相关技术背景
```

随后逐章推进，例如：

```
好的，继续第二部分，不要再用斜杠，括号也少用
```

```
好的，开始第三部分，严格参考示例的写法，公式上下连贯，一个步骤里别再分点，公式来源必须具体
```

```
好的，接下来写第四部分
```

```
非常好，现在画第六部分的两个图，用 mermaid，里面都用中文
```

### Skill 的工作方式

- 每章完成后等待反馈，再继续下一章
- 用户说"继续"或"继续第X部分"，直接写，不反复确认
- 用户说"缺点不用写七个，四五点就行"之类的调整指令，立即执行

## 各章节核心规范

### 第一章 · 背景技术
- 现有专利引用 2 篇，格式：`CN号 + 一句话核心方案`
- 缺点列 4-5 点，每点有小标题 + 1-2 句展开
- 末尾综上句自然引出本发明

### 第二章 · 技术问题
- 三个问题，每个以疑问句开头
- 结构：疑问句 → 现有方法不足 → 本发明提出……
- 整段叙述，不分小点

### 第三章 · 技术方案（最严格）
- 约 10 个步骤，每步一个公式（附编号）
- 公式须有明确来源（最小二乘、MAD、Welford 增量式等）
- 每个公式中所有符号逐一说明
- **前后参数连贯**：前步骤定义的变量，后续步骤直接引用，不重新定义
- 步骤之间须有逻辑衔接，整段叙述，不分小点

### 第四章 · 技术优点
- 4-5 点，标题用对仗或升级式句式（如"由……升级为……"）
- 每点先说现有不足，再说本发明改进，约 80-120 字

### 第五章 · 欲保护点
- 4-5 个保护点，与第三章核心步骤对应
- 每个保护点末尾明确列出"保护点："细项

### 第六章 · 附图
- 两张 Mermaid 图：总体架构图 + 技术细节图
- 全部中文，用 subgraph 分层，节点 8-12 个，不过于纵长

---

## 文风要求

- 不口语化：不用"我们"、"可以看到"、"很明显"
- 不 AI 化：不用"值得注意的是"、"综合来看"、"总的来说"
- 少括号、少引号、不用斜杠
- 正式书面，参照中文学术论文语气
- 公式有依据，参数有来源，不杜撰

---

## 适用场景

- 电力、能源、工业制造领域的方法类发明专利
- 涉及数据处理、质量评估、状态监测、特征提取等方向
- 需要含公式推导的技术方案章节
- 需要对标已有技术交底书风格进行仿写
